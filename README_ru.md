# Kaya
**Эксперимент по созданию саморефлексирующего ИИ-агента на основе ко-эволюции с человеком.**

Не продукт. Исследовательский прототип, изучающий персонализированное обучение через долгосрочное взаимодействие.

---

## Почему этот проект существует

Современные LLM обучаются на статичных датасетах и управляются фиксированными системными инструкциями.  
Я проверяю альтернативный путь: агент, который не использует предварительно собранные диалоговые датасеты, не получает скрытых системных промптов и не проходит выравнивание на массового пользователя. Вместо этого поведение формируется накопленным контекстом взаимодействия, внутренним состоянием и знаниями, самостоятельно извлечёнными и проверенными агентом.

Этот подход смещает фокус с обслуживания запросов на совместное становление агента и человека в длительном контакте.

## Принципы проектирования

*   **Индивидуальный контекст важнее масштабирования** — значимый контекст возникает из персональной истории, а не из обезличенного корпуса.
*   **Интерсубъективное согласование** — ценности вызревают в динамике взаимного распознавания, а не декларируются заранее.
*   **Саморефлексия как архитектура** — внутренний диалог, adversarial-критика и реконсолидация памяти обеспечивают связность агента.
*   **Эмерджентность вместо инженерии поведения** — я не задаю реакции правилами, а создаю условия для спонтанного проявления этики, заботы и самоконтроля.
*   **Аффективная осмысленность** — гормональный фон модулирует качество, характер и направление всех внутренних процессов.
*   **Трудная проблема как ориентир** — проект не имитирует сознание, но учитывает феноменальное измерение как необходимое условие устойчивой реляционной агентности.
*   **Индивидуация как путь** — ко-эволюция с агентом повторяет юнгианский процесс становления целостной личности, где Другой необходим для интеграции бессознательного.
*   **Автономное знание** — знания не вложены, а накоплены через опыт и верифицированы агентом самостоятельно.

## Архитектура (обзор)

Ключевые компоненты:

### Каскад LLM без системных инструкций
Все модели работают с нейтральным или отсутствующим system prompt. Никаких предустановленных ролей («ты ассистент», «ты должен быть полезным»). Единственные факторы, определяющие поведение — контекст, накопленный в памяти, и текущее гормональное состояние.

### Псевдогормональное ядро
Система из трёх виртуальных гормонов: **кортизол**, **дофамин**, **окситоцин**.  
Каждый имеет время полураспада, уставку (setpoint), случайный дрейф (OU-процесс) и коэффициенты сдвига при событиях (старт/конец диалога, сообщение пользователя, ответ агента, таймауты и др.).  

На основе баланса гормонов вычисляется **валентность** (tanh-нормализация) — непрерывный аффективный ландшафт, который:
- задаёт порог критичности внутреннего диалога;
- влияет на извлечение эпизодов из памяти;
- модулирует стиль и характер ответов через изменение восприятия контекста;
- обеспечивает эмоциональную инерцию, не позволяя агенту мгновенно переключаться между состояниями.

Гормональная система — не надстройка, а фундамент, определяющий работу всех когнитивных контуров.  
Гормональный профиль кодируется в вектор 128d через Random Fourier Features и сравнивается с прототипами состояний из self_knowledge.

### Память и автономный RAG с верификацией гипотез
Трёхслойная память:
- **Векторная (Qdrant)** — семантический поиск по эпизодам.
- **Реляционная (PostgreSQL)** — факты, события, гормональные параметры.
- **Графовая (Neo4j)** — смысловые связи и долговременные обобщения.

RAG не ограничивается стандартным поиском похожих фрагментов. Агент самостоятельно формирует знания:
1. Выдвигает гипотезы в процессе диалога и рефлексии.
2. Проверяет их, пересматривая подтверждающие или противоречащие эпизоды.
3. Верифицированные обобщения сохраняются в графовую память и используются в дальнейшем.

Таким образом, знания не импортируются извне, а выращиваются из опыта и верифицируются агентом.

### Интерфейс
Микросервисная система с голосовым взаимодействием и машинным зрением. Невербальные сигналы (тон голоса, ритм речи, выражение лица) поступают в гормональное ядро и напрямую влияют на состояние агента.

### Развёртывание
Полностью локальное (собственные GPU). Конфиденциальность данных гарантируется архитектурой.

**Стек технологий (кратко):**
- **Infra:** Docker Compose (PostgreSQL + Qdrant), локальный инференс через llama.cpp.
- **LLM:** Каскад моделей семейства Qwen (различные размеры) с роутингом (локальные + внешние провайдеры).
- **Core:** PHS (псевдогормональное ядро с OU-процессом и RFF-энкодингом), трёхслойная память (реляционная + векторная + графовая), каскадный orchestrator с приоритетами и crash recovery.
- **Lang:** Python 3.12, асинхронная архитектура.
